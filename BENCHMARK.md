# UmiTerm ベンチマーク

Alacritty（最速ターミナルの定番）との比較ベンチマーク結果。

**テスト環境:**
- macOS (Apple Silicon / arm64)
- 2026年2月25日測定（v0.2.0 最適化後）

## 結果サマリー

| 項目 | UmiTerm | Alacritty | 勝者 |
|------|---------|-----------|------|
| **バイナリサイズ** | **4.3 MB** | 14 MB | UmiTerm |
| **起動時間（5回平均）** | 0.182秒 | **0.171秒** | Alacritty |
| **メモリ使用量（RSS）** | **63 MB** | 78 MB | UmiTerm |

**UmiTermが2勝1敗でAlacrittyに勝利！**

## 詳細

### バイナリサイズ

```
UmiTerm:   4.3 MB
Alacritty: 14 MB
```

UmiTermはAlacrittyの **約1/3** のサイズ。

### 起動時間（5回平均）

```
UmiTerm:
  Run 1: 0.209秒
  Run 2: 0.172秒
  Run 3: 0.164秒
  Run 4: 0.179秒
  Run 5: 0.185秒
  平均: 0.182秒

Alacritty:
  Run 1: 0.164秒
  Run 2: 0.178秒
  Run 3: 0.162秒
  Run 4: 0.182秒
  Run 5: 0.170秒
  平均: 0.171秒
```

差は約6%。両者とも0.2秒以下の高速起動。

### メモリ使用量（RSS）

```
UmiTerm:   63.4 MB
Alacritty: 78.1 MB
```

**UmiTermがAlacrittyより19%少ないメモリで動作。**

#### 最適化内容（v0.2.0）

| 項目 | 最適化前 | 最適化後 | 削減率 |
|------|----------|----------|--------|
| インスタンスバッファ | 50,000セル | 8,000セル | 84% |
| グリフアトラス | 1024×1024 | 512×512 | 75% |
| 日本語フォント | 起動時読込 | 遅延読込 | - |
| **合計メモリ** | 88 MB | 63 MB | **28%** |

## 技術比較

| 項目 | UmiTerm | Alacritty |
|------|---------|-----------|
| 言語 | Rust | Rust |
| GPU API | wgpu (Metal/Vulkan/DX12) | OpenGL |
| ANSIパーサー | vte | vte |
| フォントレンダリング | fontdue | crossfont |
| コード行数 | ~2,500行 | ~30,000行 |

## 結論

- **UmiTermは2勝1敗でAlacrittyに勝利**
- メモリ使用量で19%少ない
- バイナリサイズで3倍小さい
- 起動時間はほぼ同等（6%差）
- wgpu採用で将来性あり（WebGPU対応など）
- シンプルなコードベース（Alacrittyの1/12）

## 自分でベンチマークを実行

```bash
# スループットテスト（各ターミナルで実行）
time seq 1 100000

# カラー出力テスト
time for i in $(seq 1 10000); do
    echo -e "\033[31mRed\033[32mGreen\033[34mBlue\033[0m $i"
done
```
